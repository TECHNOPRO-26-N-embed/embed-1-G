## 1. システム全体像

### 1-1. ブロック構成（入力 → 処理 → 出力）

> ※ Arduinoを中心に「何が入力で何が出力か」をテキストの図で整理します。

**記入例：**
```
【入力】                 【Arduino UNO R3】            【出力】
タクトスイッチ ─────→┐                         ┌──→ LED（赤・緑）
超音波センサー ─────→┤   状態管理・判定処理    ├──→ パッシブブザー
                      └─────────────────────────┘
```

**↓ 自分のシステムを書いてください**
```


【入力】                 【Arduino UNO R3】            【出力】
ジョイスティック ────→┐                         ┌──→ LEDドットマトリクス（スネーク・エサ表示）
                         │   ゲーム状態管理・       │ブザー（パッシブ/アクティブ）
                         │   スネーク制御・         │LCDディスプレイ（スコア表示）
                         │   スコア管理             │
                         └─────────────────────────┘

```

補足説明：
ジョイスティックでスネークの進行方向を操作。LEDマトリクスにゲーム画面を表示。エサ取得やゲームオーバー時にブザーで効果音。スコアはLCDに表示（任意機能）。

### 1-2. 状態遷移（ガジェットの「ふるまい」設計）

> ※ ガジェットが「どんな状態を持つか」を定義します。
> この設計が loop() の骨格になります。

**記入例：**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（センサー検知）──→ [計測中] ──（閾値超え）──→ [警告中]
   ↑                                                            │
   └────────────────────（リセットボタン）────────────────────────┘
```

**↓ 自分の状態遷移を書いてください**
```
[電源ON / 初期化]
  ↓（初期化完了）
[待機中] ──（ジョイスティック押し込み）──→ [ゲーム中] ──（ゲームオーバー判定）──→ [ゲームオーバー]
   ↑　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　│
   └────────────（ジョイスティック押し込み）────────────┘

```


**各状態の意味：**

| 状態名 | この状態のとき、何をしているか | LEDやブザーなど出力の様子 |
|:--|:--|:--|
| 待機中 | ゲーム開始待ち。ジョイスティック押し込み待機 | LEDマトリクス：タイトル表示、ブザー：無音 |
| ゲーム中 | スネーク移動・エサ取得・スコア加算 | LEDマトリクス：ゲーム画面、ブザー：エサ取得時効果音 |
| ゲームオーバー | ゲーム終了、再スタート待ち | LEDマトリクス：ゲームオーバー表示、ブザー：効果音 |

> [!TIP]
> すべての状態から「抜け出せるルート」があるか確認してください。
> 「このボタンを押せば必ずリセットに戻る」が保証されていますか？

### 2-3. タイミング設計（delay vs millis）

> ※ `delay(N)` はN ms間、**すべての処理**が止まります。
> 複数の処理を並行させたい場合は `millis()` を使います。

| 処理 | 必要な周期 | delay / millis | 判断の理由 |
|:--|:--|:--|:--|
| スネーク移動 | 200〜500ms | millis | ゲーム速度調整・他処理と並行 |
| ジョイスティック入力取得 | 10ms | millis | 入力遅延防止 |
| LEDマトリクス描画 | 50ms | millis | なめらかな表示のため |
| 効果音再生 | 随時 | millis | ゲーム進行と並行 |
| LCD表示更新 | 100ms | millis | 任意機能・他処理と並行 |

> [!CAUTION]
> `delay()` を使う場合、その間はボタン入力もセンサー読み取りも**すべて止まります**。

### 3-1. 部品一覧とピン接続

> ※ デジタルピン: D2〜D13 / アナログピン: A0〜A5
> 特殊用途: D0/D1（シリアル通信・使用禁止）、A4/A5（I2C）、D10〜D13（SPI）

| 部品名 | 型番・種類 | Arduinoピン | 入力 / 出力 | 備考 |
|:--|:--|:--|:--|:--|
| ジョイスティック | アナログジョイスティックモジュール | A0（X軸）, A1（Y軸）, D2（押し込み） | 入力 | グループ共通部品 |
| LEDドットマトリクス | MAX7219 8x8 | D10（CS）, D11（CLK）, D12（DIN） | 出力 | SPI通信 |
| パッシブブザー | - | D5 | 出力 | 効果音用 |
| アクティブブザー | - | D6 | 出力 | エサ取得音用 |
| LCDディスプレイ | 1602(I2C) | A4, A5 | 出力 | 任意機能 |
| ブレッドボード/ジャンパ線 | - | - | - | 配線用 |

**ピン競合チェック結果：** 競合なし

> [!CAUTION]
> **ピン競合チェック（必須）**
> - D0/D1 は Serial 通信専用のため使用禁止
> - I2C 機器（LCD等）は A4/A5 を使用
> - 同じピンに複数の部品を繋いでいないか確認

**ピン競合チェック結果：** 競合なし / 競合あり → 競合なし

## 4. 異常系・エラー処理の方針

|:--|:--|:--|:--|
| チャタリング | ジョイスティック押し込み時 | 前回入力から50ms未満の入力 | デバウンス処理でスキップ |
