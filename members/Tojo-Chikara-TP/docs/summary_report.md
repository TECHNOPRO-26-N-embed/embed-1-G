## 1. 何を作ったか（30秒）

<!-- ひと言で伝わるように書いてください。「○○を使って、△△するガジェットを作りました。」 -->

**ガジェット名：**
シューティングゲーム
**ひと言で説明：**
ジョイスティックでプレイヤーを動かし、流れてくる敵を避けるミニゲームを作成した。押し込み操作で開始・リトライする。

**使った部品：**
- Arduino UNO
- ジョイスティックモジュール
- 8×8 LEDマトリクス
- ブザー

**動いたもの：**
- ジョイスティック入力でプレイヤーが8×8上を移動

## 4. 一番苦労したこと、どう乗り越えたか（30秒）

<!-- ここが発表の山場です。1つだけに絞って、ストーリーで書いてください -->

**何が起きたか：**
LEDマトリクスが上手く表示されず、コードが動作していないのか回路側に問題があるのか、原因がわからなかった。

**どう対処したか：**
部品単体テストに戻り、LEDマトリクスのみを動かす最小コードで動作確認を行った。
配線を1本ずつ順に見直した。


**そこから何がわかったか：**
全部つないで一気に動かすよりも、最小構成で動作確認を行い、段階的に統合していく手順が最も確実な解決策であると分かった。わざわざ単体テスト、結合テストと分割して検証していくことの大事さを実際に体験することができた。
