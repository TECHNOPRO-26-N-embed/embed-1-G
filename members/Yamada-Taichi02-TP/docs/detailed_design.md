```
【タイマー（millis()用）】（basic_design.md 2-3 から転記）

【その他のフラグ・カウンター】
lastChatteringTime : unsigned long = 0; // ボタン用チャタリング防止時刻

```

## 2. 各関数の詳細設計

### `setup()` — 初期化処理

```
【処理の流れ】
1. ピンモードを設定する
   - PIN_BUTTON     → INPUT_PULLUP; // ボタン入力
   - PIN_LED_GREEN  → OUTPUT;       // 緑LED出力
   - PIN_MOTOR_1    →OUTPUT;        // モーター制御ピン1
   - PIN_MOTOR_2    →OUTPUT;        // モーター制御ピン2
   
2. 起動確認（任意）: 緑LEDを1秒点灯して消灯
```

---

### `loop()` — メインループ

> ※ loop() は「状態ごとに何をするか」だけ書く。細かい処理は各関数に任せる。

```
【処理の流れ】

＜毎ループ実行すること＞
  - 現在時刻を取得: now = millis()
  - now - lastMillis_Joystick >= 50 のとき readJoystick() を実行する
  - now - lastMillis_Button >= 50 のとき readButton() を実行し、押下確定時は setSpeedMode() を実行する

＜currentState が Idle（待機中）のとき＞
  - joystickInput を監視する（デッドゾーン: LOW=132 / HIGH=892）
  - joystickInput がデッドゾーン外なら → currentState = Move


＜currentState が Move（動作中）のとき＞
  - updateOutput() を実行する


＜currentState が ERROR（エラー）のとき＞
  - モーターを停止し、エラー表示をする
  - リセット条件を満たしたら → currentState = Idle

```

### `updateOutput()` — joystickInputに応じてモーター・LED・LCDを更新する

**basic_design.md 2-2 との対応：** 出力更新

**引数：** `なし`

**戻り値：** `void`

```
【処理の流れ】

【エラー・異常ケース】

```

## 3. 重要ロジックの詳細設計

### 3-1. チャタリング防止（デバウンス処理）

> ※ ボタンを使う場合は必ず設計してください。

```
【考え方】
  ボタンが押されたとき、50ms 以内の連続入力は「同じ1回の押下」として無視する。

【処理の流れ】
  1. ボタンのデジタル値を読む（digitalRead）
  2. 前回確定した時刻（lastChatteringTime）からの経過時間を計算する
  3. 経過時間 < CHATTERING_DELAY（例: 50ms）→ 無視する
  4. 経過時間 ≥ CHATTERING_DELAY → ボタンの状態として確定する
  5. lastChatteringTime を更新する

【必要な変数（Section 1 に追加済みか確認）】
  lastChatteringTime : unsigned long  // 前回確定した時刻
  CHATTERING_DELAY   : int = 50       // チャタリング判定時間（ms）
```

---

### 3-2. millis() を使ったタイマー管理

```
【考え方】
  delay() は使わず、処理ごとに「前回実行時刻」を保持して50ms周期で実行する。
  これにより、ジョイスティック入力・ボタン入力・モーター制御を並行して監視できる。

【処理の流れ（本システム）】
  1. ループ先頭で now = millis() を取得する。
  2. ジョイスティック監視:
     - 条件: now - lastMillis_Joystick >= 50
     - 実行: readJoystick()
     - 更新: lastMillis_Joystick = now
  3. ボタン監視:
     - 条件: now - lastMillis_Button >= 50
     - 実行: readButton()（チャタリング判定付き）、押下確定時は setSpeedMode()
     - 更新: lastMillis_Button = now
  4. モーター制御:
     - 条件: now - lastMillis_Motor >= 50
     - 実行: updateOutput()
     - 更新: lastMillis_Motor = now
  5. 条件を満たさない処理はそのループでは実行せず、次ループで再判定する。

【周期設計（basic_design.md 2-3 との対応）】
  - ジョイスティック入力監視: 50ms（millis）
  - ボタン入力監視: 50ms（millis）
  - モーター制御: 50ms（millis）
```

### Q1: 実装上の問題確認

> 「この詳細設計書に書いた関数と処理フローをもとに Arduino でコードを書きます。バグになりやすい箇所・処理の抜け・型の問題はありますか？」

**AIの回答（要約）：**
- チャタリング変数名が `lastChatteringTime` と `lastDebounceTime` で混在している。  
- `updateOutput` の引数定義が不明瞭で、関数仕様に揺れがある。

**対応した内容：**
- `updateOutput()` を引数なしに修正 
- チャタリング関連を `lastChatteringTime` / `CHATTERING_DELAY` に統一。  
- Section 3-3 にデッドゾーン判定と出力対応を追記。

### Q2: 単体テスト仕様の確認

> 「Section 5 の単体テスト仕様書で、各関数の動作が正しく検証できていますか？テストが不足している項目や、境界値テストが必要な箇所を教えてください。」

**AIの回答（要約）：**
